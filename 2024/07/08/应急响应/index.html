Webshell查杀

基于时间配合日志分析

看被攻击之前的日志，注意访问路径、访问方法（除了登录上传，正常请求不会有post请求）、访问的ua头（比如sqlmap有独特的ua头）、访问的状态码。看到奇怪的行为全局搜索ip，跟踪他到底做了什么事情

比如在日志中显示他对index.php进行了post请求，然后使用了sqlmap扫描。我们可以访问index路径，看看是什么页面，比如是个登录框，那我们就可以用sqlmap测试下这里是不是有问题

基于漏洞配合日志分析

自己先去测一测这个网站，看哪里有问题。根据自己攻击的路径或者请求信息之类的信息去日志里搜索有没有人也这么请求过

基于后门配合日志分析

通过扫描后门的名字，来去在日志里定位后门是什么时候上传的

Webshell查杀工具：

  • 阿里伏魔https://ti.aliyun.com/#/webshell（查杀效果好，但是是网页版本，要一个个文件上传，一般源码都有很多个文件，一个个上传不现实）
  • 河马https://n.shellpub.com/（客户端，安装在服务器上可以自己查杀）

内存马查杀

  • .NET：https://github.com/yzddmr6/ASP.NET-Memshell-Scanner（放到服务器上，浏览器访问这个脚本）

  • PHP：常规后门查杀检测后，中间件重启后删除文件即可

  • JAVA：河马，脚本java-memshell-scanner-master（放到服务器上，浏览器访问这个脚本，无法判断可以dump下来分析）

后门查杀

windows

常规后门

比如cs或者msf，上面河马之类的只能查有毒的文件，这个就不一定可以查到了，后门查杀可以使用一些检测网络连接的工具，比如火绒剑

这里用cs测试下，看火绒剑网络这一块，数字签名文件大概率是安全的，除非是免杀做的特别好的，所以主要看未知文件，这里就很明显了，还可以看到对外连接的ip，这个就是攻击ip了。

也可以在系统里选择开启监控，来看这个进程做了什么请求

权限维持后门

自启动：

REG ADD "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /V "backdoor" /t REG_SZ /F /D "C:\cs.exe"

隐藏账户：

使用pchunter

net user znskiw$ znskiw@#X123 /add

映像劫持：

REG ADD "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\notepad.exe" /v debugger /t REG_SZ /d "C:\Windows\System32\cmd.exe /c calc"（点击notepad时，启动calc）

登录：

REG ADD "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /V "Userinit" /t REG_SZ /F /D "C:\shell.exe"

判断方法cpu拉满

windows

可以在火绒剑网络里看这个进程外连的ip，也可以找到挖矿病毒的配置文件的域名，来查矿池。可以把ip或者域名丢到微步或者奇安信威胁情报中心看看是否有问题

所以看windows的日志，windows日志在这

linux

使用top命令查看cpu占用率，使用find / -name 进程名查看进程的位置，记录下病毒植入时间，查看配置文件，看看域名，去威胁情报社区搜索

因为挖矿病毒一般还会植入个定时任务，让病毒可以一直执行，所以要先删掉定时任务

  • crontab -l 或者在/var/spool/cron/crontabs查看定时任务（有些会隐藏文件，要看到仔细一点）

  • 删除挖矿病毒

  • 通过时间、漏洞、webshell来定位日志等方法看看攻击者是什么漏洞进来的，修复漏洞，删除后门

  • 重启服务器
