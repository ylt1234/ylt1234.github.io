这题有点离谱，要猜出源码

先补充几个知识点：
Mysql中的||是or的意思：

  • 0 || 1 = 1
  • 2 || 3 = 1
  • 0 || 0 = 0

而||还有连接符的作用：
eg：

相当于：

当 sql_mode 设置了 PIPES_AS_CONCAT 时，|| 就是字符串连接符，相当于CONCAT() 函数
当 sql_mode 没有设置 PIPES_AS_CONCAT 时 （默认没有设置），|| 就是逻辑或，相当于OR函数

select 和 select *的区别：
select * 语句取出表中的所有字段
假如表 Table1 有：a,b,c,d,e 五列
select * from Table1 就相当于 select a,b,c,d,e from Table1 显示所有列的记录。
select a,b,c from table1 就只显示 a,b,c 三列的记录。

构造payload:

放回句子中就是：select 1;set sql_mode=pipes_as_concat;select 1 || flag from Flag。set sql_mode=pipes_as_concat将||的作用从or变成了连接符，相当于和。句子可以等价成select 1;select 1 and flag from Flag,输出的结果是先输出select 1，再将select 1和select flag拼起来输出
